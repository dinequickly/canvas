## 2026-03-18

- Investigated why board-level embedded iframes look blurry until interacted with. The main likely cause is that Cerebrus board shapes are rendered inside an `HTMLContainer` that applies `transform: scale(...)` for non-primary emphasis states (`0.985` / `0.96` / `0.94`), which can make nested iframe content rasterize softly until the shape becomes the active focused item.
- A second contributing factor is that Cerebrus iframe blocks disable pointer interaction outside editing mode (`pointerEvents: none`, `tabIndex: -1`, `zIndex: -1`), so the first interaction changes the editing/focus state around the embed even though the iframe itself is not otherwise styled with blur filters.
- Fixed the board blur issue for live Cerebrus document surfaces by keeping document-rendered shapes at `scale(1)` even when they are de-emphasized in folder/map view. Map-card previews still use the old scale-based emphasis, but live documents and their nested iframes now preserve crisp rasterization on first render.
- Fixed the remaining board wash-out on live embeds: the same wrapper had still been applying de-emphasis opacity (`0.94` / `0.7`) to document-rendered shapes, which made standalone `VideoEmbed` / `IFrame` surfaces look greyed out until interaction. Live document surfaces now stay fully opaque on the board while summary map cards still use the old emphasis fade.
- Rewired the production bottom dock to the real default-agent pipeline. The dock now submits `requestId` / `uiOrigin` tagged agent requests through `/stream` instead of directly calling the Cerebrus helper, and its compact response panel is driven by a new app-level `AgentAppDockManager`.
- Added two new main-agent handoff actions, `generateCerebrus` and `startOrchestration`, alongside the existing `spawnArtifact` action. The model can now explicitly choose typed in-canvas Cerebrus generation, deliverable artifact generation, orchestration workflows, or plain direct canvas actions from the same working-mode action set.
- Added dock-aware run metadata to agent requests and chat history (`requestId`, `uiOrigin`) so dock runs can be summarized cleanly and stale async action results do not overwrite newer dock prompts.
- Added dock-native reporting for each handoff path: Cerebrus runs report model/op counts and raw operations JSON, artifact runs report created artifact metadata plus open affordance, orchestration runs report the live root agent id and descendant status, and generic direct-action runs fall back to grouped action summaries from chat history.
